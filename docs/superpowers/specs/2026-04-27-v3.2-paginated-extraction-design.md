# v3.2 Paginated Extraction — Design Specification

**Date**: 2026-04-27  
**Target Release**: v3.2.0  
**Estimated Implementation**: 2-3 days (14 hours)  
**Goal**: Achieve 100% extraction success rate on large design system files (50+ pages)

---

## Executive Summary

v3.1 Sync Mode achieves 78% success rate across 18 tested design systems. The remaining 22% (4 systems) fail due to timeout on files with 50+ pages. v3.2 introduces **paginated extraction** to eliminate these timeouts and achieve 100% success rate.

**Key improvement**: 8.5/10 maturity → **9.0/10 maturity**

---

## Problem Statement

### Current Limitation (v3.1)

Single-pass extraction algorithm exceeds MCP transport timeout (~60 seconds) on large files:

| System | Pages | Result |
|---|---:|---|
| Carbon Design (IBM) | 54 | Timeout, minimal extraction |
| Elastic UI | 65 | Timeout, minimal extraction |
| Untitled UI PRO | 113 | Timeout, minimal extraction |
| Flowbite Pro | 208 | Timeout, minimal extraction |

**Root cause**: `use_figma` attempts to traverse all pages in a single call, accumulating data until timeout.

**Current workaround**: `--component ComponentName` flag for targeted sync (not ideal for system-wide sync).

### Success Criteria (v3.2)

- ✅ All 18 tested systems extract successfully (100% success rate)
- ✅ No timeouts on files up to 200+ pages
- ✅ Backward compatible (small files unchanged)
- ✅ User-visible progress reporting
- ✅ Resume capability on interruption
- ✅ No performance regression on small files

## Solution: Paginated Extraction with Incremental Snapshots

### Architecture Overview

```
User runs: /design-nexus sync [URL]
          ↓
[Phase 1-Sync] Extract current Figma state
          ↓
    Detect page count
          ↓
    ┌─────────────────────────────────┐
    │  < 50 pages?                    │
    │  → Use single-pass (v3.1 logic) │──→ Continue to Phase 2
    └─────────────────────────────────┘
          ↓
    ┌─────────────────────────────────────┐
    │  ≥ 50 pages?                        │
    │  → Switch to paginated mode (v3.2)  │
    └─────────────────────────────────────┘
          ↓
    Calculate batches (10 pages/batch)
          ↓
    Check for existing partial snapshots
          ↓
    Resume from last completed batch (if any)
          ↓
    ┌─────────────────────────────────────┐
    │  For each remaining batch:          │
    │  1. Extract via use_figma           │
    │  2. Save partial snapshot           │
    │  3. Display progress                │
    └─────────────────────────────────────┘
          ↓
    Merge all partial snapshots
          ↓
    Save final baseline snapshot
          ↓
    Clean up .partial/ directory
          ↓
[Phase 2-Sync] Detect drift (existing logic)
```

### Key Design Decisions

**1. Automatic detection with progress reporting**
- User doesn't need to know/care about batching
- Progress indicators prevent "is it frozen?" concern
- No new flags or complexity for users

**2. Incremental snapshots with merge**
- Each batch saved independently (no corruption risk)
- Resume capability (can recover from any interruption)
- Debuggable (partial snapshots show exactly what was extracted)

**3. Batch size: 10 pages**
- Balances timeout safety vs. number of batches
- 10 pages ≈ 5-8 seconds per batch (well under 60s timeout)
- 200-page file = 20 batches ≈ 2-3 minutes total

**4. Resume on interruption**
- Partial snapshots persist in `.partial/` directory
- On restart, detect existing batches and continue
- No wasted work (previous batches reused)

## Detailed Design

### 1. File Structure

```
.design-nexus/
├── snapshots/
│   ├── [fileKey].json              # Final baseline (same as v3.1)
│   └── .partial/                   # NEW: Temporary partial snapshots
│       ├── [fileKey]-batch-0.json  # Pages 0-9
│       ├── [fileKey]-batch-1.json  # Pages 10-19
│       ├── [fileKey]-batch-2.json  # Pages 20-29
│       └── ...
```

**Lifecycle:**
- Partial snapshots created during extraction
- Merged into final snapshot on completion
- `.partial/` directory cleaned up after successful merge
- `.partial/` persists on interruption for resume

**Gitignore:**
- `.partial/` directory is gitignored (ephemeral, like `drift-reports/`)
- Final snapshots remain version-controlled

### 3. Batch Detection Logic

**Location**: Phase 1-Sync, `extractCurrentState()` function

**Pseudocode:**
```javascript
async function extractCurrentState(fileKey, targetComponent) {
  const pageCount = figma.root.children.length;
  const PAGINATION_THRESHOLD = 50;
  const BATCH_SIZE = 10;
  
  // Route decision
  if (targetComponent) {
    // Component-specific sync always uses single-pass
    console.log(`🎯 Extracting component: ${targetComponent}`);
    return await extractSingleComponent(fileKey, targetComponent);
  }
  
  if (pageCount < PAGINATION_THRESHOLD) {
    // Small files use existing v3.1 single-pass
    console.log(`📄 Extracting ${pageCount} pages (single-pass mode)`);
    return await extractSinglePass(fileKey);
  }
  
  // Large files trigger paginated extraction
  console.log(`📦 Large file detected (${pageCount} pages)`);
  console.log(`   Switching to paginated extraction mode...\n`);
  return await extractWithPagination(fileKey, pageCount, BATCH_SIZE);
}
```

**Constants:**
- `PAGINATION_THRESHOLD = 50` pages
- `BATCH_SIZE = 10` pages per batch

**Rationale:**
- 50-page threshold chosen based on validation data (largest successful single-pass: Elastic UI at 65 pages timed out)
- 10-page batch size provides 6x safety margin (10 pages ≈ 8s, timeout at 60s)

### 4. Batch Extraction Flow

**Core algorithm:**

```javascript
async function extractWithPagination(fileKey, pageCount, batchSize) {
  const totalBatches = Math.ceil(pageCount / batchSize);
  const partialDir = `.design-nexus/snapshots/.partial/`;
  
  // Step 1: Check for existing partial snapshots (resume support)
  const existingBatches = await findExistingBatches(fileKey, partialDir);
  const startBatch = existingBatches.length;
  
  if (startBatch > 0) {
    console.log(`📂 Resuming from batch ${startBatch + 1} (found ${startBatch} existing batches)\n`);
  }
  
  // Step 2: Process remaining batches
  for (let batchIndex = startBatch; batchIndex < totalBatches; batchIndex++) {
    const startPage = batchIndex * batchSize;
    const endPage = Math.min((batchIndex + 1) * batchSize - 1, pageCount - 1);
    
    // Extract this batch
    const startTime = Date.now();
    const partial = await extractBatch(fileKey, startPage, endPage, batchIndex);
    const duration = ((Date.now() - startTime) / 1000).toFixed(1);
    
    // Save partial snapshot atomically
    await savePartialSnapshot(partial, partialDir);
    
    // Display progress
    displayBatchProgress(
      batchIndex,
      totalBatches,
      startPage,
      endPage,
      Object.keys(partial.components).length,
      duration
    );
  }
  
  // Step 3: Merge all partial snapshots
  const merged = await mergePartialSnapshots(fileKey, totalBatches, partialDir);
  
  // Step 4: Save final snapshot
  await saveSnapshot(merged, `.design-nexus/snapshots/${fileKey}.json`);
  
  // Step 5: Clean up .partial/ directory
  await cleanupPartialSnapshots(fileKey, partialDir);
  
  return merged;
}
```

**Per-batch extraction:**

```javascript
async function extractBatch(fileKey, startPage, endPage, batchIndex) {
  const components = {};
  let variables = null;
  
  // Extract components from pages [startPage...endPage]
  for (let i = startPage; i <= endPage && i < figma.root.children.length; i++) {
    const page = figma.root.children[i];
    await figma.setCurrentPageAsync(page);
    
    // Reuse v3.1 component extraction logic
    const pageComponents = await extractPageComponents(page);
    Object.assign(components, pageComponents);
  }
  
  // Extract variables ONCE (file-level, not page-level)
  if (batchIndex === 0) {
    variables = await extractVariables(fileKey);
  }
  
  // Return partial snapshot data
  return {
    version: '3.2.0',
    fileKey,
    fileName: figma.root.name,
    fileUrl: `https://www.figma.com/design/${fileKey}`,
    batchIndex,
    pageRange: {
      start: startPage,
      end: endPage,
      pageNames: figma.root.children.slice(startPage, endPage + 1).map(p => p.name)
    },
    timestamp: new Date().toISOString(),
    components,
    variables: variables || { collections: [], variables: [] },
    metadata: {
      componentCount: Object.keys(components).length,
      extractionDuration: 0 // filled by caller
    }
  };
}
```

**Key points:**
- Each batch is an independent `use_figma` call
- Pages loaded sequentially within each batch
- Variables extracted only in batch 0 (file-level data)
- Component IDs are globally unique (no deduplication needed)

### 6. Snapshot Merging

**Strategy: Incremental merge with validation**

```javascript
async function mergePartialSnapshots(fileKey, totalBatches, partialDir) {
  console.log(`\n🔗 Merging ${totalBatches} batches...`);
  const startTime = Date.now();
  
  // Initialize merged snapshot
  const merged = {
    version: '3.2.0',
    timestamp: new Date().toISOString(),
    figma: {
      fileKey,
      fileName: '',
      fileUrl: ''
    },
    components: {},
    variables: {
      collections: [],
      variables: []
    },
    metadata: {
      extractionMode: 'paginated',
      batchCount: totalBatches
    }
  };
  
  // Read and merge all partial snapshots
  for (let i = 0; i < totalBatches; i++) {
    const partialPath = `${partialDir}${fileKey}-batch-${i}.json`;
    const partial = JSON.parse(await readFile(partialPath));
    
    // Validate batch index matches
    if (partial.batchIndex !== i) {
      throw new Error(
        `Batch index mismatch: expected ${i}, got ${partial.batchIndex}\n` +
        `Partial snapshot may be corrupted.`
      );
    }
    
    // Merge components (simple object spread)
    Object.assign(merged.components, partial.components);
    
    // Merge variables (only from batch 0)
    if (i === 0) {
      merged.variables = partial.variables;
      merged.figma.fileName = partial.fileName;
      merged.figma.fileUrl = partial.fileUrl;
    }
  }
  
  // Calculate final metadata
  merged.metadata.totalComponents = Object.keys(merged.components).length;
  merged.metadata.totalVariables = merged.variables.variables.length;
  merged.metadata.pageCount = totalBatches * 10; // approximate
  
  const duration = ((Date.now() - startTime) / 1000).toFixed(1);
  console.log(`   ✓ (${duration}s)`);
  
  return merged;
}
```

**Deduplication:**
- Component IDs are Figma node IDs (globally unique)
- `Object.assign()` naturally deduplicates (same ID = overwrite)
- Variables are file-level (not page-level), extracted only in batch 0

**Merge complexity:**
- Linear O(n) where n = total components
- Fast even for large systems (2-3 seconds for 4,000+ components)

### 7. Resume Logic

**Scenario: Extraction interrupted at batch 5 of 12**

**Detection:**
```javascript
async function findExistingBatches(fileKey, partialDir) {
  const batches = [];
  let batchIndex = 0;
  
  while (true) {
    const path = `${partialDir}${fileKey}-batch-${batchIndex}.json`;
    
    // Check if batch file exists
    if (!await fileExists(path)) {
      break;
    }
    
    try {
      // Validate JSON structure
      const content = await readFile(path);
      const parsed = JSON.parse(content);
      
      // Validate batch index matches expected
      if (parsed.version === '3.2.0' && parsed.batchIndex === batchIndex) {
        batches.push(parsed);
        batchIndex++;
      } else {
        // Corrupted or misnamed file
        console.warn(
          `⚠️  Invalid batch file at index ${batchIndex}: ` +
          `expected batchIndex=${batchIndex}, got ${parsed.batchIndex || 'undefined'}\n` +
          `   Discarding and restarting from this batch.`
        );
        await deleteFile(path);
        break;
      }
    } catch (error) {
      // Invalid JSON
      console.warn(
        `⚠️  Corrupted JSON at batch ${batchIndex}: ${error.message}\n` +
        `   Discarding and restarting from this batch.`
      );
      await deleteFile(path);
      break;
    }
  }
  
  return batches;
}
```

**Resume execution:**
```javascript
// In extractWithPagination():
const existingBatches = await findExistingBatches(fileKey, partialDir);
const startBatch = existingBatches.length;

if (startBatch > 0) {
  console.log(
    `📂 Resuming from batch ${startBatch + 1} ` +
    `(found ${startBatch} existing batches)\n`
  );
}

// Process only remaining batches
for (let i = startBatch; i < totalBatches; i++) {
  // ...extract and save batch i...
}
```

**Resume guarantees:**
- Atomic writes: Partial snapshots written to temp file, then renamed
- Validation: Each batch file validated before reuse
- Corruption handling: Invalid batches discarded, restart from that point
- No data loss: All completed batches reused

**User experience:**

### 8. Error Handling

**Network timeout during batch:**
```javascript
async function extractBatch(fileKey, startPage, endPage, batchIndex) {
  const MAX_RETRIES = 3;
  let lastError;
  
  for (let attempt = 1; attempt <= MAX_RETRIES; attempt++) {
    try {
      // Attempt extraction with Figma API retry logic
      return await callFigmaWithRetry(() => 
        extractBatchInternal(fileKey, startPage, endPage, batchIndex)
      );
    } catch (error) {
      lastError = error;
      
      if (attempt < MAX_RETRIES) {
        const delay = Math.pow(2, attempt) * 1000; // 2s, 4s, 8s
        console.warn(
          `   ⚠️  Batch ${batchIndex + 1} failed ` +
          `(attempt ${attempt}/${MAX_RETRIES}): ${error.message}`
        );
        console.log(`   ↻ Retrying in ${delay/1000}s...\n`);
        await sleep(delay);
        continue;
      }
    }
  }
  
  // All retries exhausted
  throw new Error(
    `Batch ${batchIndex + 1} (pages ${startPage}-${endPage}) ` +
    `failed after ${MAX_RETRIES} attempts:\n\n` +
    `${lastError.message}\n\n` +
    `Partial snapshots saved through batch ${batchIndex}.\n` +
    `Run sync again to resume from batch ${batchIndex + 1}.`
  );
}
```

**Merge failure:**
```javascript
async function mergePartialSnapshots(fileKey, totalBatches, partialDir) {
  try {
    // ... merge logic ...
    return merged;
  } catch (error) {
    throw new Error(
      `Failed to merge partial snapshots:\n` +
      `${error.message}\n\n` +
      `Partial snapshots preserved in:\n` +
      `${partialDir}\n\n` +
      `You can manually inspect or delete them.`
    );
  }
}
```

**Cleanup failure (non-fatal):**
```javascript
async function cleanupPartialSnapshots(fileKey, partialDir) {
  try {
    // Delete all batch files
    for (let i = 0; i < 999; i++) {
      const path = `${partialDir}${fileKey}-batch-${i}.json`;
      if (await fileExists(path)) {
        await deleteFile(path);
      } else {
        break;
      }
    }
    console.log(`🧹 Cleaned up partial snapshots\n`);
  } catch (error) {
    // Non-fatal: Log warning but don't fail the sync
    console.warn(
      `⚠️  Warning: Could not clean up partial snapshots:\n` +
      `   ${error.message}\n` +
      `   You can manually delete: ${partialDir}${fileKey}-batch-*.json\n`
    );
  }
}
```

```javascript
// Entry point (Phase 1-Sync)
async function extractCurrentState(fileKey, targetComponent) {
  const pageCount = figma.root.children.length;
  
  if (pageCount >= 50 && !targetComponent) {
    return await extractWithPagination(fileKey, pageCount, 10);
  }
  
  return await extractSinglePass(fileKey, targetComponent);
}

// Paginated extraction
async function extractWithPagination(fileKey, pageCount, batchSize) {
  const totalBatches = Math.ceil(pageCount / batchSize);
  const partialDir = `.design-nexus/snapshots/.partial/`;
  
  // Resume support
  const existingBatches = await findExistingBatches(fileKey, partialDir);
  const startBatch = existingBatches.length;
  
  // Extract remaining batches
  for (let i = startBatch; i < totalBatches; i++) {
    const startPage = i * batchSize;
    const endPage = Math.min((i + 1) * batchSize - 1, pageCount - 1);
    
    const partial = await extractBatch(fileKey, startPage, endPage, i);
    await savePartialSnapshot(partial, partialDir);
    displayBatchProgress(i, totalBatches, startPage, endPage, ...);
  }
  
  // Merge and finalize
  const merged = await mergePartialSnapshots(fileKey, totalBatches, partialDir);
  await saveSnapshot(merged, `.design-nexus/snapshots/${fileKey}.json`);
  await cleanupPartialSnapshots(fileKey, partialDir);
  
  return merged;
}

// Per-batch extraction
async function extractBatch(fileKey, startPage, endPage, batchIndex) {
  const components = {};
  let variables = null;
  
  for (let i = startPage; i <= endPage; i++) {
    await figma.setCurrentPageAsync(figma.root.children[i]);
    const pageComponents = await extractPageComponents(figma.root.children[i]);
    Object.assign(components, pageComponents);
  }
  
  if (batchIndex === 0) {
    variables = await extractVariables(fileKey);
  }
  
  return {
    version: '3.2.0',
    fileKey,
    batchIndex,
    pageRange: { start: startPage, end: endPage },
    timestamp: new Date().toISOString(),
    components,
    variables: variables || { collections: [], variables: [] }
  };
}

// Merge partial snapshots
async function mergePartialSnapshots(fileKey, totalBatches, partialDir) {
  const merged = {
    version: '3.2.0',
    components: {},
    variables: { collections: [], variables: [] }
  };
  
  for (let i = 0; i < totalBatches; i++) {
    const partial = await readPartialSnapshot(fileKey, i, partialDir);
    Object.assign(merged.components, partial.components);
    
    if (i === 0) {
      merged.variables = partial.variables;
    }
  }
  
  return merged;
}
```
