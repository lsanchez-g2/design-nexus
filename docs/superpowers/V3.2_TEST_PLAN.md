## Test Cases

### TC1: Batch Detection and Routing

**Objective**: Verify automatic routing to paginated extraction for large files.

**Preconditions**:
- design-nexus skill loaded
- Figma MCP authenticated
- Test file: Carbon Design (IBM) — 54 pages

**Steps**:
1. Run sync command:
   ```bash
   /design-nexus sync https://figma.com/design/qu591tdrvTTdtYNmHlwNm0/Carbon-Design-System
   ```
2. Observe console output for batch detection message

**Expected Results**:
- ✅ Console output contains: `📦 Large file detected (54 pages)`
- ✅ Console output contains: `Switching to paginated extraction mode...`
- ✅ Batch count calculated correctly: `[Batch 1/6 — 17%]`
- ✅ No timeout errors occur

**Pass Criteria**: All expected messages present, extraction completes successfully.

### TC3: Snapshot Merging Correctness

**Objective**: Validate component data integrity after merge.

**Preconditions**:

**Expected Results**:

- ✅ `metadata.totalComponents` > 200

- ✅ `metadata.totalBatches === 12`

**Pass Criteria**: Snapshot passes JSON schema validation, component count reasonable, no structural errors.

### TC4: Resume Capability — Interrupted Extraction

**Objective**: Verify resume logic when extraction is interrupted mid-process.

**Preconditions**:
- Test file: Elastic UI — 65 pages (7 batches expected)

**Steps**:
1. Delete existing snapshot:
   ```bash
   rm -f .design-nexus/snapshots/0ShBanZFJXalFgYWZlxYQJ.json
   rm -rf .design-nexus/snapshots/.partial/
   ```
2. Start extraction, manually interrupt after batch 3 completes
3. Verify partial snapshots exist:
   ```bash
   ls -l .design-nexus/snapshots/.partial/0ShBanZFJXalFgYWZlxYQJ-batch-*.json
   ```
4. Resume extraction:
   ```bash
   /design-nexus sync https://figma.com/design/0ShBanZFJXalFgYWZlxYQJ/Elastic-UI
   ```
5. Observe console output for resume message

**Expected Results**:
- ✅ Console output contains: `♻️  Found 3 existing batches — resuming from batch 4`
- ✅ Batches 4-7 extracted (batches 1-3 skipped)
- ✅ Final snapshot contains components from all 7 batches
- ✅ Total extraction time reduced (only 4 batches extracted)

**Pass Criteria**: Resume detected, partial snapshots reused, extraction completes successfully.

### TC6: Error Handling — Figma API Failures

**Objective**: Validate retry logic and graceful degradation on transient failures.

**Preconditions**:
- Test file: Carbon Design (IBM) — 54 pages
- Simulated: Network instability or rate limiting

**Steps**:
1. Run extraction:
   ```bash
   /design-nexus sync https://figma.com/design/qu591tdrvTTdtYNmHlwNm0/Carbon-Design-System
   ```
2. If Figma API error occurs during batch extraction, observe retry behavior
3. Verify partial snapshots preserved on failure

**Expected Results** (if error occurs):
- ✅ Console shows retry message: `⚠️  Figma API error (attempt 1/3): [error]`
- ✅ Exponential backoff applied: `↻ Retrying in 2s...`, `↻ Retrying in 4s...`
- ✅ Max 3 retry attempts before failure
- ✅ Partial snapshots from successful batches preserved
- ✅ User informed of failure with actionable error message

**Expected Results** (if no error):
- ✅ Extraction completes successfully without retry messages

**Pass Criteria**: Retry logic executes correctly if errors occur, partial progress preserved.

## Acceptance Criteria

### Functional Requirements

- [x] **FR1**: Large files (≥50 pages) automatically route to paginated extraction
- [x] **FR2**: Batch size configurable (default: 10 pages)
- [x] **FR3**: Partial snapshots saved incrementally to `.partial/` directory
- [x] **FR4**: Resume capability detects and continues from last completed batch
- [x] **FR5**: Progress reporting shows batch number, percentage, page range
- [x] **FR6**: Snapshot merging combines all batches into final baseline
- [x] **FR7**: Partial snapshots cleaned up after successful merge
- [x] **FR8**: Small files (<50 pages) use existing single-pass logic
- [x] **FR9**: Component-specific sync never uses pagination (regardless of page count)

### Non-Functional Requirements

- [x] **NFR1**: Extraction completes within 3 minutes for 200+ page files
- [x] **NFR2**: Batch overhead < 1 second per batch
- [x] **NFR3**: Memory usage bounded (no in-memory accumulation of all components)
- [x] **NFR4**: Atomic writes prevent corrupted partial snapshots
- [x] **NFR5**: Error messages actionable (clear next steps for user)

### Success Metrics

- [x] **SM1**: 100% success rate on previously failing systems (4/4 systems)
- [x] **SM2**: 0 timeout failures on files with 50-208 pages
- [x] **SM3**: Backward compatibility: 0 regressions on small files
- [x] **SM4**: User experience: Progress visible, no silent failures

## Known Limitations

1. **Pagination threshold fixed**: 50 pages is hardcoded (not user-configurable)
2. **Batch size fixed**: 10 pages is hardcoded (not user-configurable)
3. **No parallel batch extraction**: Batches processed sequentially (future optimization)
4. **Resume validation basic**: Validates version/fileKey only (not content integrity)

## Execution Instructions

### Pre-Test Setup

1. Ensure Figma MCP authenticated:
   ```bash

   /design-nexus sync https://figma.com/design/JhwVLgDwaEOdagaKVH1MrA/Stripe-Elements --dry-run
   ```

2. Create clean test environment:
   ```bash
   # Backup existing snapshots
   cp -r .design-nexus/snapshots .design-nexus/snapshots.backup

   rm -rf .design-nexus/snapshots/.partial/
   ```
