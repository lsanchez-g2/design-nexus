1. **Property Structure**: Ant Design uses separate boolean/variant properties (Danger, Ghost) where shadcn/ui uses single variant property
2. **Variant Naming**: Uses "Type" instead of "variant", "medium" instead of "default"
3. **Non-Standard Variants**: Includes "dashed" variant not present in Western design systems
4. **Token Architecture**: 0% variable usage (fully hardcoded) vs shadcn's token-first approach
5. **Touch Targets**: All sizes violate WCAG 2.2 AA (< 44px) vs shadcn's accessibility-first design
6. **State Coverage**: Missing focus state vs shadcn's comprehensive state definitions

**Positive Observations**:

1. **State Granularity**: Includes "active" and "animating" states (more comprehensive than some systems)
2. **Shape Variants**: Explicit circle/square shapes for icon-only buttons
3. **Component Scale**: 205 variants = comprehensive coverage of use cases
4. **Consistent Naming**: All 205 variants follow consistent property/value naming pattern

**Recommendation for Designers**:

If migrating from Ant Design → shadcn/ui:
- Consolidate Danger/Ghost into variant property
- Rename Type → variant, medium → default
- Increase touch targets to 44×44px minimum
- Add focus state with visible ring
- Migrate to Figma variables (3-layer architecture)
